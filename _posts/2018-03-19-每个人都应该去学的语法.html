最近在写产品文档，如行云流水般滔滔不绝，多亏了好用的 Markdown 工具。同样的体量，要是搁 Word 里，起码得写一个多礼拜吧。


不得不膜拜下它的创始人：

Markdown 是一种轻量级的「标记语言」，是由美国的约翰·格鲁伯（John Gruber）和亚伦·斯沃茨（Aaron Swartz）创建的，亚伦·斯沃茨是一位很有名的极客。


Markdown 的目标是：实现「易读易写」。


暂且抛开文档的内容，毕竟内容跟人的思维有直接关系，而不是用什么工具去写。


这里所说的「易读易写」主要是针对文档格式来说。


先说说「易读性」

一份好的文档，可读性必然很高，而格式起到了至关重要的作用，就像代码规范一样，乱七八糟的缩进，会大大降低代码的可读性。


那么什么样格式的文档可读性会很高呢？


首先，不能喧宾夺主，永远要记住格式是为文字服务的。


好的格式应该是帮助读者理解文档的要义，比如重点部分会有标注或者粗体表示，他人的表述会用到引用格式，一目了然就能区分出来。而不是套用很多眼花缭乱的格式，或许会吸引了眼球，但真正重要的文字就被忽视了。


其次，结构清晰，层次分明，一眼就能看清文档的脉络。


就像很多人打字不喜欢打标点符号，要么懒得打，要么是学古人，再或者是彰显个性。我始终不太习惯，毕竟中文的方块字不加标点，堆在一起，要断个句还是挺费劲的。文档也是如此，章节段落不分明，就像是一盘散沙，可读性就会非常差。


再者，就是好看了。


缺失的字体、奇怪的底色、杂乱的对齐，一上来就给人留了个坏印象，说不定内容也会因此大打折扣。现在做什么都讲究颜值，爱美之心人皆有之，赏心悦目总是好的。


这些都能做到，那至少文档的易读性至少能过关了。


那么，为了达到「易读」，是否「易写」？


如果是用 Word 来写，强大的功能当然能够支撑，但排版也要花去不少功夫，还记得去学校打印论文的时候，被打印店妹纸嘲笑排版太弱。


有了 Markdown，一切都非常简单，虽然复杂的语法还是有一些的，不过日常使用，基本上记住*、#、>就好了，分别代表粗体、标题、引用。再多就是表格、图片之类的了，熟悉HTML语法的同学更是方便，直接标签写起来就好了。


最重要的是 Markdown 不会停止响应！然后再暗戳戳地问你「是否要恢复自动保存的版本？」，好的，结果恢复到不知道哪个版本去了(¬_¬) 


我现在用的这款写作工具 Ulysses 也支持 Markdown 语法，所以在写上面那段话的时候，从*开始就被标注出来了。貌似微信公众号排版还不支持 Markdown，希望能早日实现。


从刚开始用 Markdown 到现在，市面上能找到的工具几乎都试过，Mac下的不用说了，有好几款做的都很好。难为 Windows 平台，有时候想找到个 Mac 同款比大海捞针一样还难，好在 Typora 有出 Windows 版本，再加上 pandoc 的组合，能轻松导出成各种格式。


贴几张图感受下：


Typora 跟其他主流的 Markdown 工具不同的是，它没有做分屏显示，即左边是源码模式，右边是预览模式，二者合二为一，写完换行就是预览效果。


Markdown 也许是为程序员而生的，但并不代表只有程序员能用，简单到人人都能用，用 Markdown 写博客，写简历的大有人在，导出成 HTML 之后看着绝对高大上，毕竟写了那么多的 CSS 在里面😂


要说因为 Markdown 爱上写文档的，我绝对相信，写着写着就很嗨，不用去管那些乱七八糟的对齐、纠错，摆在你面前的就是文字，心无旁骛地写好一字一句就好了，剩下的完全不用操心。


用 Typora 的一句 slogan结束吧

Simple, yet Powerful
